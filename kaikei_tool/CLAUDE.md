# 会計大将インポートCSV生成ツール - 作業手順書

このファイルはClaude Code（CC）への指示書です。
事務所PCでCCを起動すると、このファイルを自動的に読み込みます。

---

## このツールの全体像

```
【入力】                    【処理】                     【出力】
通帳PDF ─────→                                 ┌→ import_日時.csv （会計大将にインポート）
請求書PDF ───→  Python仕訳エンジン  ──→ │
賃金台帳PDF ─→  （ルールブック参照）        └→ review_日時.csv （要確認リスト）
```

---

## フォルダ構成

```
kaikei_tool/
├── main.py              ← メインスクリプト（これを実行する）
├── config.py            ← 共通設定
├── requirements.txt     ← 必要なライブラリ一覧
├── modules/
│   ├── pdf_reader.py    ← PDF読み取り共通処理
│   ├── bank_parsers.py  ← 銀行別通帳PDF解析（★カスタマイズ対象）
│   ├── master.py        ← 科目マスタ管理
│   ├── rulebook.py      ← ルールブック管理（GASから移植済み）
│   ├── csv_exporter.py  ← 会計大将26列CSV出力
│   └── journal_engine.py ← 仕訳判定エンジン
└── clients/
    └── 顧問先名/
        ├── rulebook.txt     ← ルールブック辞書（自動生成）
        ├── 過去仕訳/         ← 過去の仕訳エクスポートCSV
        ├── 当月資料/         ← 通帳PDF、請求書PDF等
        ├── マスタ/           ← 科目リスト、補助科目等
        └── output/          ← 生成されたCSV
```

## 事務所PCでの初回セットアップ手順

### 前提条件
- Python 3.8以上がインストールされていること
- CCがインストールされていること

### 手順

CCに以下を伝えてください：

「kaikei_toolフォルダに移動して、pip install -r requirements.txt を実行してください」

CCがライブラリのインストールを行います。

## 顧問先ごとの初回準備

### 1件ずつ作成する場合

CCに以下のように伝えてください：

「山田商事の顧問先フォルダを作成してください。python main.py --init 山田商事 を実行して」

すると以下のフォルダが作成されます：
```
clients/山田商事/
├── 過去仕訳/    ← ここに会計大将から出力した仕訳CSVを入れる
├── 当月資料/    ← ここに当月の通帳PDFや請求書PDFを入れる
├── マスタ/      ← 事業者情報.json（テンプレート自動生成）
└── output/     ← 仕訳CSVがここに出力される
```

### CSVから一括作成する場合

顧問先リストのCSVファイルを使って、フォルダを一括作成できます。

CCに以下のように伝えてください：

「顧問先リスト.csv から顧問先フォルダを一括作成してください。python main.py --batch-init 顧問先リスト.csv を実行して」

#### CSVファイルの書き方

**パターン1: 顧問先名だけの1列CSV**
```
山田商事
佐藤建設
田中医院
```

**パターン2: 詳細情報つき5列CSV**
```
顧問先名,インボイス事業者,インボイス番号,会計期間開始,会計期間終了
山田商事,TRUE,T1234567890123,2025/04/01,2026/03/31
佐藤建設,FALSE,,2025/01/01,2025/12/31
田中医院,○,T9876543210987,2025/04/01,2026/03/31
```

- ヘッダー行があっても自動でスキップされます
- Shift_JIS（Excel保存）でもUTF-8でも読めます
- インボイス事業者は TRUE/FALSE または ○/× で指定できます
- 既に同名フォルダがある顧問先はスキップされます

### 準備するファイル

1. **過去仕訳CSV**（必須）
   - 会計大将から「仕訳データのCSVエクスポート」で出力したファイル
   - 26列のCSV形式
   - 1年分以上あると精度が上がる
   - `clients/山田商事/過去仕訳/` に入れる

2. **当月の通帳PDF**
   - ネットバンキングからダウンロードした通帳PDFや取引明細PDF
   - ファイル名の先頭に科目コードを付けると自動認識される
     例: `131_常陽銀行普通.pdf`、`132_みずほ当座.pdf`
   - `clients/山田商事/当月資料/` に入れる

3. **請求書PDF**（あれば）
   - ファイル名に「請求」を含めること
     例: `請求書_202603.pdf`

4. **賃金台帳PDF**（あれば）
   - ファイル名に「賃金」または「給与」を含めること
     例: `賃金台帳_202603.pdf`

## 月次の仕訳CSV生成手順

CCに以下のように伝えてください：

「山田商事の仕訳CSVを生成してください。python main.py 山田商事 を実行して」

### 出力されるファイル

1. **import_日時.csv** - 会計大将にインポートするファイル（Shift_JIS）
2. **review_日時.csv** - 要確認リスト付き（Excelで開ける）

### review_日時.csv の見方

上部に「要確認取引」一覧があります：
- **理由が「ルールブックに一致するパターンなし」** → 新しい取引先。正しい科目を教えてください
- **理由が「源泉税対象」** → 報酬科目と金額内訳を確認してください
- **理由が「借入金返済」** → 元本・利息の内訳を確認してください

## CCへのPDFカスタマイズ依頼の仕方

通帳PDFは銀行ごとにフォーマットが違うため、初回は調整が必要です。

### 対応済み銀行一覧

以下の銀行は `modules/bank_parsers.py` に専用パーサーが用意されています：

| 銀行名 | ファイル名の例 |
|--------|---------------|
| 常陽銀行 | `131_常陽銀行普通.pdf` |
| 筑波銀行 | `131_筑波銀行普通.pdf` |
| ゆうちょ銀行 | `134_ゆうちょ銀行.pdf` |
| 茨城県信用組合 | `131_茨城県信組.pdf` |
| 水戸信用金庫 | `131_水戸信金.pdf` |
| 足利銀行 | `131_足利銀行普通.pdf` |

ファイル名や中身に銀行名が含まれていれば自動で判定されます。
上記以外の銀行でも、汎用パーサーで基本的な読み取りを試みます。

### 通帳PDFの初回調整（★重要★）

各銀行のパーサーには `【★事務所PCで要調整★】` というマークがついています。
事務所PCで実際の通帳PDFを使って調整する必要があります。

CCに以下のように伝えてください：

「clients/山田商事/当月資料/131_常陽銀行普通.pdf を読み取って、
modules/bank_parsers.py の常陽銀行の解析関数を、
この通帳のフォーマットに合わせて調整してください」

CCが実際のPDFの列構造を確認し、パーサーを自動調整します。
一度調整すれば、同じ銀行の通帳は今後も正しく読み取れます。

### 請求書PDFの場合

「clients/山田商事/当月資料/請求書_202603.pdf を読み取って、
modules/pdf_reader.py の請求書PDF解析関数を調整してください」

### 新しい銀行を追加したい場合

「○○銀行の通帳PDFのパーサーを bank_parsers.py に追加してください。
clients/山田商事/当月資料/131_○○銀行.pdf を参考にしてください」

## ルールブックの仕組み

ルールブック（rulebook.txt）は過去仕訳から自動生成される辞書です。
「この摘要なら、この科目」というパターンを学習します。

### 処理フロー
1. 通帳のカタカナ摘要（例: "トウキヨウデンリヨク"）を正式摘要（"東京電力 電気代"）に変換
2. 口座コード + 出金/入金 + 摘要名 で科目を特定
3. 源泉税対象の取引先なら諸口仕訳（報酬+預り金+消費税）を自動生成
4. 借入金返済なら諸口仕訳を自動生成
5. どれにも一致しなければ「174 仮払金」にして要確認リストに追加

### ルールブックの更新
仕訳CSV生成のたびに自動更新されます。
手動でrulebook.txtを編集しても構いません。
