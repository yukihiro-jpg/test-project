# 会計大将インポートCSV生成ツール - 作業手順書

このファイルはClaude Code（CC）への指示書です。
事務所PCでCCを起動すると、このファイルを自動的に読み込みます。

---

## このツールの全体像

```
【入力】                       【処理】                        【出力】
銀行CSV ────→ ┐                                     ┌→ import_日時.csv（会計大将インポート）
銀行PDF ────→ ├→ 入力正規化                          ├→ review_日時.csv（要確認リスト+メモ）
請求書PDF ──→ ├→ ルールブック照合                     ├→ 士業報酬_顧問先名.csv（蓄積型）
賃金CSV/PDF → ┘   売掛金消込/借入金/源泉税 自動処理  └→ ルールブック自動更新
```

**設計方針: CSV優先・PDFフォールバック**
CSVが取れる入力は全てCSVで処理（精度100%）。PDFは従来通りの解析。

## 顧問先ごとの初回準備

### 顧問先フォルダの作成

```
python main.py --init 山田商事
python main.py --batch-init 顧問先リスト.csv  # CSVから一括作成
```

### 準備するファイル

1. **過去仕訳CSV**（必須）
   - 会計大将「仕訳エクスポート」で出力した26列CSV
   - 1年分以上で精度向上
   - → `clients/山田商事/過去仕訳/` に配置

2. **銀行取引データ**（毎月）
   - **CSV推奨**: インターネットバンキングからCSVダウンロード
   - PDFも対応（フォールバック）
   - ファイル名: `131_常陽銀行.csv` or `131_常陽銀行.pdf`
   - → `clients/山田商事/当月資料/` に配置

3. **売上請求書PDF**（あれば）
   - ファイル名に「請求」を含める
   - → 発生主義で売掛金/売上の仕訳を自動生成
   - → 入金時に自動消込（手数料差引対応）

4. **賃金台帳**（あれば）
   - **CSV推奨**: マネーフォワード/弥生給与/楽しい給与計算のCSV出力
   - PDFも対応
   - ファイル名に「賃金」「給与」を含める

5. **返済予定表CSV**（借入金がある場合）
   - 列: 日付, 元本, 利息（ヘッダー自動認識）
   - → `clients/山田商事/返済予定表/` に配置

## 月次の仕訳CSV生成手順

```
python main.py 山田商事
```

### 自動で行われること

1. **入力ファイルの自動分類**: CSV/PDFを自動判定、銀行/請求書/賃金を自動振り分け
2. **重複排除**: 前回処理済みの取引は自動スキップ
3. **売掛金消込**: 請求書の売掛金と入金を自動突合（手数料パターン対応）
4. **借入金返済**: 返済予定表から元本・利息を自動分割
5. **士業報酬**: 源泉税10.21%を自動計算、報酬CSVに蓄積
6. **バリデーション**: 貸借一致・科目コード存在・日付範囲・重複を自動チェック
7. **ルールブック自動更新**: 処理結果を自動で学習

### 出力ファイル

| ファイル | 説明 |
|---------|------|
| `import_日時.csv` | 会計大将にインポートするファイル（Shift_JIS） |
| `review_日時.csv` | 要確認リスト+処理メモ付き（UTF-8、Excelで開ける） |
| `士業報酬_顧問先名.csv` | 士業報酬の蓄積データ（処理のたびに追記） |

## 複合仕訳の自動処理

### 売掛金消込（発生主義+入金時消込）

- 請求書PDF → 売掛金/売上の発生仕訳を自動生成
- 通帳入金時に未消込売掛金と自動突合
  - 完全一致 → 売掛金消込
  - 手数料差引（330,495,550,660円等〜1,100円） → 複合仕訳（手数料+消込）
  - 不一致 → 仮払金+メモで要確認

### 借入金返済

- `返済予定表/` フォルダにCSVを配置すると自動で元本・利息を分割
- 予定表なし → review行き（元本・利息の確認が必要）

### 士業への支払い（源泉税）

- 源泉税10.21%を自動計算
- ルールブックから報酬科目を自動特定
- `士業報酬_顧問先名.csv` に処理データを蓄積

## ルールブックの仕組み

ルールブック（rulebook.txt）は過去仕訳から自動生成・自動更新される辞書です。
手動フィードバック不要。仕訳CSV生成のたびに自動で学習します。

### 処理フロー
1. 通帳のカタカナ摘要を正式摘要に変換
2. 口座コード + 出金/入金 + 摘要名 で科目を特定
3. 入金時は売掛金消込を試行（手数料差引対応）
4. 源泉税対象なら諸口仕訳（報酬+預り金+消費税）
5. 借入金返済なら返済予定表参照で元本・利息分割
6. いずれも不一致→174仮払金+要確認リストに追加
