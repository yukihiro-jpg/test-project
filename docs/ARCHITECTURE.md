## 月次処理フロー

```
[1] 税理士がダッシュボードで「当月の資料作成」をクリック
     │
     ▼
[2] MJS会計大将の4種類のCSVをアップロード
     │
     ▼
[3] 文字コード判定（Shift-JIS→UTF-8）＆ パース
     │
     ▼
[4] 帳票間の整合性バリデーション
     ├─ 試算表 vs 元帳の月次集計突合
     ├─ 試算表 vs 推移試算表の累計値突合
     └─ 不一致があればエラー表示で停止
     │
     ▼
[5] Firestore にデータ保存（顧問先×対象月）
     │
     ▼
[6] 資料自動生成（7セクション）
     ├─ 前月コメント引継ぎ（宿題の自動表示）
     ├─ 異常値検知（閾値: 前月比±10%/前年同月比±20%）
     ├─ ベンチマーク引当（業種×規模）
     └─ AI叩き台コメント生成（Gemini）
     │
     ▼
[7] ブラウザで確認 → 税理士がAIコメントを編集・確定
     │
     ▼
[8] PDF出力 → 印刷
     │
     ▼
[9] 社長と打合せ → 各ページ下部のコメント欄に入力（PC）
     │
     ▼
[10] コメント入りPDF再出力 → Gmail API で社長へ送信
     │
     ▼
[11] 次月宿題の生成（タグ付きコメントを翌月に引継ぎ）
```

## データモデル概要（Firestore）

```
clients/{clientId}
  ├─ profile: ClientProfile（社長プロファイル）
  └─ reports/{reportId}
       ├─ sections: ReportSection[]
       ├─ validation: ValidationResult
       └─ comments/{commentId}

benchmark/{fiscalYear}/industries/{industryCode}
  └─ scales/{capitalScale}
       └─ indicators: BenchmarkData[]
```

詳細は `docs/DATA_MODEL.md` を参照。

## 主要モジュール

### `src/lib/parsers/`
MJS会計大将のCSVパーサー群。4種類の帳票それぞれに対応。
Shift-JIS対応、和暦変換、集計行の階層判定を含む。

### `src/lib/anomaly/`
異常値検知ロジック。前月比・前年同月比の閾値超過を自動検出し、
該当科目について元帳を参照して変動要因を特定する。

### `src/lib/ai/`
Gemini API クライアント。社長プロファイル、前月コメント、
ベンチマークデータをコンテキストに含めて叩き台コメントを生成。

### `src/lib/pdf/`
Puppeteer で HTML テンプレートを A4 PDF にレンダリング。
印刷用途を考慮したフォント埋め込み・ページ番号付与を行う。

### `src/lib/excel/`
ExcelJS でネイティブ Excel を生成。数字セルは数値型、
グラフは Excel ネイティブのグラフとして埋め込む。
